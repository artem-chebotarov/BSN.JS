## Описание проекта

bsn.js (Bootstrap Service Notification) — это легковесная и доступная обёртка над модальными окнами Bootstrap. Этот плагин был создан для полноценной замены нативных браузерных функций alert, confirm и prompt.
Ключевая особенность: Полная поддержка программ экранного доступа (Screen Readers). Это гарантирует, что незрячие и слабовидящие пользователи смогут взаимодействовать с вашими диалогами без каких-либо препятствий.
 
 ---

## API и Подключение

Для использования библиотеки просто подключите скрипт на вашу HTML-страницу:

```html
<script src="./src/bsn.js"></script>
```

---

### Важное примечание:

При подключении плагина в глобальной области видимости регистрируется переменная $gn. Этот стандарт заимствован из GiannaFramework. Внутри объекта $gn.bsn хранятся все методы уведомлений. Если вы планируете использовать другие модули GiannaFramework, они также будут доступны через объект $gn.*.

---

## Основные методы


### $gn.bsn.alert()

Метод выводит модальное уведомление, которое требует обязательного подтверждения от пользователя.

#### Синтаксис:

```js
$gn.bsn.alert({
    title: "",    // Заголовок окна
    message: "",  // Текст сообщения
    ok_text: "",  // Текст кнопки подтверждения
    app: "",      // Дополнительная метка (например, название модуля)
    oneShowStart: (e) => {}, // событие show.bs.modal
    oneShowEnd: (e) => {}, // Событие shown.bs.modal
    oneHideStart: (e) => {}, // Событие hide.bs.modal,
    oneHideEnd: (e) => {}, // Событие hidden.bs.modal
    ok: () => {}  // Callback-функция после нажатия кнопки
});
```

Метод возвращает результат выполнения callback или undefined.

---

### $gn.bsn.confirm()

Метод запрашивает подтверждение действия. Возвращает логическое значение (boolean). Поскольку метод асинхронный, рекомендуется использовать его с await.

#### Синтаксис:

```js
let result = await $gn.bsn.confirm({
    title: "",    // Заголовок
    message: "",  // Сообщение вопроса
    app: "",      // Подпись приложения
        oneShowStart: (e) => {}, // событие show.bs.modal
    oneShowEnd: (e) => {}, // Событие shown.bs.modal
    oneHideStart: (e) => {}, // Событие hide.bs.modal,
    oneHideEnd: (e) => {}, // Событие hidden.bs.modal
    ok_text: "",  // Текст кнопки "Да"
    no_text: "",  // Текст кнопки "Нет"
    ok: () => {}, // Действие при согласии
    no: () => {}  // Действие при отказе
});
```

---

### $gn.bsn.prompt()

Метод запрашивает текстовый ввод у пользователя. Возвращает введенные данные или undefined при отмене.

#### Синтаксис:

```js
let value = await $gn.bsn.prompt({
    title: "",    // Заголовок
    message: "",  // Инструкция для пользователя
    input: "",    // Описание поля (title для скринридеров)
    value: "",    // Предзаполненный текст в поле
    focus: true, // Audofocus
    app: "",      // Подпись модуля
        oneShowStart: (e) => {}, // событие show.bs.modal
    oneShowEnd: (e) => {}, // Событие shown.bs.modal
    oneHideStart: (e) => {}, // Событие hide.bs.modal,
    oneHideEnd: (e) => {}, // Событие hidden.bs.modal
    ok_text: "",  // Кнопка подтверждения
    no_text: "",  // Кнопка отмены
    ok: () => {}, // Callback подтверждения
    no: () => {}  // Callback отмены
});
```

 ---

### $gn.bsn.toast()

Метод выводит всплывающие уведомления (тосты), которые могут исчезать автоматически.

#### Синтаксис:

```js
$gn.bsn.toast({
    title: "",    // Заголовок уведомления
    message: "",  // Текст сообщения
    type: "",     // Стиль (например: success, danger, warning, info)
    delay: 5000,  // Время отображения в миллисекундах
        oneShowStart: (e) => {}, // событие show.bs.toast
    oneShowEnd: (e) => {}, // Событие shown.bs.toast
    oneHideStart: (e) => {}, // Событие hide.bs.toast,
    oneHideEnd: (e) => {}, // Событие hidden.bs.toast
    ok_text: "",  // Текст на кнопке (если нужна)
    ok: () => {}  // Действие при нажатии на кнопку в тосте
});
 ```

## Технические примечания

- Автономность шаблонов: Вам не нужно вручную создавать HTML-разметку для окон. Все структуры встроены в класс и генерируются автоматически.
- Производительность: Шаблон каждого метода добавляется в DOM-дерево только один раз при первом вызове. В дальнейшем класс просто переиспользует существующие элементы, обновляя контент.
- Оптимизация: Плагин не оставляет «мусорных» элементов в дереве после завершения работы.
- Гибкость: Все параметры конфигурации опциональны. Если их не указывать, будут использованы стандартные значения на английском языке.
- Инклюзивность: Библиотека полностью протестирована на корректность передачи фокуса и озвучивания содержимого программами экранного доступа.
